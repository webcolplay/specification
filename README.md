## 规范更新步骤指南

 1. 提issues给项目管理者；
 2. 团队成员一起讨论这个issues设计规范的可行性；
 3. 不通过，直接关闭issues；
 4. 通过，新建分支修改规范，将新分支merge request到master分支，新生成一个版本，并关闭相应issues；
